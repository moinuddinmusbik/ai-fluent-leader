## The Big Idea

Stop asking which is better — Claude Code or Codex. The real question is what each tool is training you to become as an agent operator. Claude trains the *steer* habit: stay close to the work, converse through ambiguity, correct in real time. Codex trains the *dispatch* habit: write a bounded assignment, stand back, demand proof. These are two fundamentally different ways of managing AI labor — and the skill of 2026, Nate argues, is building **agent literacy** across both.

## Why It Matters Now

Coding agents arrived first in software because code has built-in proof ("does the code run or not?"). The same agent-management habits are now spreading into research, sales notes, spreadsheets, legal summaries — every kind of knowledge work that lives in files and messages. Non-technical leaders should force themselves into this debate, because the mental models you build today will shape how you imagine what agents can do for years. "A chatbot answers and an agent takes a job" — most organizations are still optimizing for chatbots while the agent transition is already under way.

## His Argument

- **Interfaces train behavior, not just features:** Mac vs. Windows didn't just compete on specs — each taught users what a computer was *for*. Claude and Codex are doing the same for agents. The interface you inhabit for the next six months will shape your mental model.
- **Claude = Cockpit (steer):** Stay close to the model, steer in real time. Use for fuzzy problems — taste, design judgment, ambiguity, writing, architecture. Serious users build a CLAUDE.md context file, use hooks, run plan-mode before edits, connect tools via MCP servers.
- **Codex = Operations Desk (dispatch):** Multiple parallel threads, visible work queue, inspectable outputs, a separate 5.5 review model auditing intent before permitting action. Computer use and background automations extend it beyond coding.
- **Decision rule:** Claude when the shape of the question is hard; Codex when the work can be written as a bounded assignment; both when stakes are high (one plans, other critiques; one implements, other reviews).

## The Contrarian Edge

- "I don't think the honest answer is Claude wins or Codex wins. They're pulling the future in different directions."
- Nate's arc: "It was Claude first and now it's Codex" — not a dismissal, but a phase in the learning curve
- "Prompting is far too small a word for what we're doing here. We're doing agent loop management."
- Real leaderboard: which interface makes permissions obvious, proof hard to forget, repeated work into durable skills

## What It Means For You

1. **Classify the work before you pick the tool.** Fuzzy/taste/judgment → Claude's cockpit. Bounded/assignable/checkable → Codex's operations desk.
2. **Build proof requirements into every agent run.** Demand receipts — files, logs, diffs, sources — before trusting what comes back.
3. **Use both when stakes are high.** Agent loop management: one plans, other critiques; one implements, other reviews.
4. **Name your failure mode.** Claude: conversation theater (feel close but aren't). Codex: completion theater (run looks done but isn't).

## Mistake to Avoid

Treating tool choice as a benchmark decision. The tool you spend the most time in trains your mental model of what agents can do — and that model outlasts any leaderboard. Developers who switch report genuine cognitive friction. Choose deliberately.
