## Core Message

Claude Opus 4.8 introduced dynamic workflows to Claude Code — they look like skills, subagents, and /goal but are fundamentally different. Dynamic workflows are Claude's ability to generate and adapt its own tool-use sequence at runtime, without a hard-coded pipeline. Claude Code can now replan mid-task when it encounters unexpected environments.

## What He Covers

- **What dynamic workflows are:** Runtime-generated tool-use sequences — Claude decides which tools to call and in what order based on mid-execution observations. No pre-defined pipeline.
- **vs. slash commands:** Fixed pre-defined sequences. Use for known, repeatable processes where speed and auditability matter.
- **vs. subagents:** Separate Claude instances with own context. Use for genuinely parallel workstreams.
- **vs. /goal:** Static pre-written plan for open objective. Use when path is unpredictable at start but plan is fixed once set.
- **When to reach for each:** Fixed → slash command. Parallel → subagents. Open objective → /goal. Adaptive mid-task → dynamic workflow.
- **Live demo:** Real build where dynamic workflows let Claude pivot after encountering unexpected file structure.

## As an AI-Fluent Leader — What To Do This Week

1. **Identify one task that breaks your slash commands.** Where the path isn't predictable — that's your dynamic workflow candidate.
2. **Let Claude plan before it executes.** Prompt: "Before starting, outline the tools and sequence you'll use, and flag where you might need to adjust."
3. **Build your pattern decision matrix.** Fixed → slash command. Parallel → subagents. Open objective → /goal. Adaptive path → dynamic workflow.
4. **Watch for unnecessary replanning.** Add: "Complete in at most N tool calls" if Claude over-iterates.

## Mistake to Avoid

**Treating dynamic workflows as a replacement for all patterns.** More powerful but slower and harder to audit than a well-written slash command. Use for genuine novelty, not as a default.
