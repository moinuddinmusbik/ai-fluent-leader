## The Big Idea

On July 1, Fable 5 came back online after 18 days offline — but the real signal was what the outage revealed. The leaders who were never disrupted owned the harness and routed by job type. In a world where any model can be throttled, repriced, capped, or pulled behind a policy filter overnight, the skill is not picking a favorite model: it is routing the right intelligence to the right job, so that when one pipe closes, you already know where the work goes next.

## The Framework: Job-First Routing

Named [[job-first-routing]] — a 3-tier + specialist system. Define the job before picking the model.

- **Tier 1 — Daily Driver:** The model you reach for before the task is fully defined. Must handle messy, mixed, judgment-heavy work. Breadth is the requirement; harness quality matters as much as model quality.
- **Tier 2 — Cheap Workhorse (GLM 5.2):** Center-of-distribution work — familiar shapes, repeatable, easy to review: meeting summaries, standard PowerPoints, landing page drafts, CRM cleanups, routine synthesis, familiar code. GLM 5.2 often beats Claude on these at a fraction of the cost.
- **Tier 3 — Frontier Model (Claude, ChatGPT):** [[fable-style-problem|Fable-style problems]] — when shape is unclear, generalized intelligence is required, or judgment is load-bearing. "There is no substitute for a frontier model" on novel tasks.
- **Tier 4 — Specialists:** Images (Flux, Zimage, Grok Image), Video (LTX → Seed Dance API → Grok for clips), Live Web (Grok for X posts). Learn the job first, walk to the model name from there.

## How It Works

- Ask job questions first — the answers route to a tier before any model name enters the conversation.
- Familiar vs. novel: known shape + easy review → Cheap Workhorse; unclear shape, high stakes, first-time domain → Frontier.
- Harness quality is not separable from model quality. Gemini's intelligence is strong but harness is difficult; GLM 5.2 launched Z.AI as a native harness — Chinese open-source is now pursuing harness quality.
- Test before committing: run actual inputs through any candidate model before declaring it a daily driver.

## Where It Applies

- **Individual inside a company:** Permission is the first filter. Use routing logic to pick your daily driver. Failure on real work = a business case for a better model, not a prompting problem.
- **Small team leader:** Don't build a 20-model system. Pick 5 recurring artifacts, draw a straight line to value for each. Team AI fluency is the constraint.
- **Small business owner:** Full flexibility — use it to match specialists to heavy workloads; don't expand the stack past what the team can absorb.

## Receipts

- **Coinbase:** Increasing token usage while decreasing costs via routing to open-source (GLM, Kimmy).
- **Cursor:** Built on Kem; migrated to pre-trained model for savings.
- **Lindy:** Moving serious traffic to DeepSeek.
- **Shopify & Airbnb:** Qwen-style routing.
- **Microsoft:** Testing into DeepSeek architecture.
- **Fable 5 return terms:** Usage caps, credit model, filter to weaker model — the model you built on is not guaranteed to be the same model tomorrow.
- **Gemini:** Strong intelligence, weak harness — de-emphasized in his own stack.

## What It Means For You

1. **Map your five recurring artifacts.** Assign each to a tier: familiar → GLM 5.2, messy/novel → frontier, specialized → specialist.
2. **Stop paying frontier prices for center-of-distribution work.** Familiar shapes belong on a cheap workhorse.
3. **Own the harness, not the model.** Build around a harness you control. Re-routing should be five minutes, not a crisis.
4. **Test with real inputs before declaring a daily driver.** Run the actual work, not hypotheticals.

## Mistake to Avoid

Picking a model before defining the job. Nate: "I am seeing a trap that a lot of people are falling into where they confuse the act of picking a model with getting real work done." Fable's 18-day outage exposed exactly who had made this mistake.
