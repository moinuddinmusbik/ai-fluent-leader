      <h2 style="margin:0 0 12px;font-size:15px;text-transform:uppercase;letter-spacing:1px;color:rgb(26,26,46)">The Big Idea</h2>
      <p style="margin:0;font-size:16px;line-height:1.7;color:rgb(34,34,34)">The AI race is no longer about which foundation model scores highest on benchmarks. It's about which organizations can <em>move fastest</em> — observing real usage, drawing the right inference, and shipping a response before the window closes. Anthropic spotted developers using Claude to sort expense receipts; ten days later, Claude Cowork existed. That 10-day loop is Nate's evidence for a structural claim: operational velocity is now as large a competitive moat as model capability itself. The framework that makes that velocity possible — and that any team can adopt — is what he calls the <strong>Open Engine</strong>.</p>

    <!-- THE FRAMEWORK -->
    <tr><td style="padding:24px 36px 20px;border-bottom:1px solid rgb(238,238,238)">
      <h2 style="margin:0 0 12px;font-size:15px;text-transform:uppercase;letter-spacing:1px;color:rgb(26,26,46)">The Framework: Open Engine</h2>
      <ul style="margin:0;padding-left:20px;font-size:15px;line-height:1.8;color:rgb(51,51,51)">
        <li style="margin-bottom:10px"><strong>The problem it solves:</strong> Most multi-AI workflows have a hidden bottleneck — the human who hand-carries state between tools. "That's not seven jobs," Nate writes. "It's one job crossing seven systems. And every time it crosses, you carry the state." The integration layer is you. Open Engine removes you from that role.</li>
        <li style="margin-bottom:10px"><strong>What it is:</strong> A shared task queue plus a seven-part task record that travels with each job — source, decision trail, limits, and what the next tool is allowed to touch. One AI's finished output becomes the next AI's structured task, not a raw chat dump.</li>
        <li style="margin-bottom:10px"><strong>The receipt vocabulary:</strong> After each agent run, a short accountability log (the "receipt") records what was done, what changed, and what was skipped — so "done" stops meaning "now go audit it yourself."</li>
        <li style="margin-bottom:10px"><strong>The one-loop audit:</strong> Nine questions that identify your single most painful inter-tool handoff and convert it into an agent-claimable task in roughly 30 minutes. Entry point is deliberately small.</li>
        <li style="margin-bottom:10px"><strong>Prompt Mode vs. Work Mode distinction:</strong> Prompt mode asks an AI for an answer; Work Mode hands it a task record and asks for documented results — a fundamentally different relationship that makes agent accountability possible.</li>
      </ul>
    </td></tr>

    <!-- HOW IT WORKS -->
    <tr><td style="padding:24px 36px 20px;border-bottom:1px solid rgb(238,238,238)">
      <h2 style="margin:0 0 12px;font-size:15px;text-transform:uppercase;letter-spacing:1px;color:rgb(26,26,46)">How It Works</h2>
      <ul style="margin:0;padding-left:20px;font-size:15px;line-height:1.8;color:rgb(51,51,51)">
        <li style="margin-bottom:10px"><strong>Step 1 — Map the commute:</strong> Identify the single cross-tool job that hurts most (e.g., transcript → argument extraction → draft → review → publish). Use the one-loop audit's nine questions to spec it.</li>
        <li style="margin-bottom:10px"><strong>Step 2 — Build the task record:</strong> Copy-paste Nate's seven-part template into whichever AI you already use. The record carries: goal, source material, decision constraints, prior context, output format, completion criteria, and receipt stub.</li>
        <li style="margin-bottom:10px"><strong>Step 3 — Let agents claim, pause, resume:</strong> Because the task record travels with the work, any agent can pick up where another left off — the context doesn't live in a private chat that disappears.</li>
        <li style="margin-bottom:10px"><strong>Step 4 — Read the receipt:</strong> After the run, the receipt surfaces what changed, what was skipped, and what needs human review — making delegation legible without requiring a full audit trail re-read.</li>
      </ul>
    </td></tr>

      <ul style="margin:0;padding-left:20px;font-size:15px;line-height:1.8;color:rgb(51,51,51)">
        <li style="margin-bottom:10px"><strong>Solo operators running multiple AI tools:</strong> If you use Claude for reasoning, Codex for code changes, and ChatGPT for drafts — you're already doing the handoff manually. Open Engine automates the in-between.</li>
        <li style="margin-bottom:10px"><strong>Managers whose team AI work stays siloed:</strong> When one person's best AI session vanishes when they close the tab, the team can't learn from it. The task record makes individual AI work legible to the next agent — or the next person.</li>
        <li style="margin-bottom:10px"><strong>Organizations building the operational moat:</strong> Companies where "the integration layer is you" are one architectural decision away from building the kind of 10-day velocity Anthropic demonstrated with Claude Cowork.</li>
        <li style="margin-bottom:10px"><strong>Adjacent systems in the space:</strong> OpenClaw, Hermes, and Symphony offer variants; Open Engine's differentiator is that it's copy-paste deployable with no custom engineering required.</li>

      <ol style="margin:0;padding-left:20px;font-size:15px;line-height:1.8;color:rgb(51,51,51)">
        <li style="margin-bottom:12px"><strong>Map your most painful cross-tool handoff this week.</strong> Don't start with a system — start with the one job that crosses the most tools most often. Use the one-loop audit's nine questions. If you can answer them, you have your 30-minute build target.</li>
        <li style="margin-bottom:12px"><strong>Reframe your AI strategy question.</strong> The executive question is no longer "which model should we standardize on?" It's "how fast can our team observe, infer, and ship?" Set a 10-day internal velocity benchmark for AI-enabled workflow changes.</li>
        <li style="margin-bottom:12px"><strong>Measure the integration layer cost.</strong> Estimate how many person-hours per week your team spends carrying state between AI tools manually. That number is the ROI case for building the task-record infrastructure — before any model upgrade delivers a return.</li>
        <li style="margin-bottom:12px"><strong>Distinguish prompt mode from work mode in team AI policies.</strong> Prompt mode (ask for an answer) and work mode (hand off a task with a record) require different review, accountability, and handoff protocols. Most AI policies cover only prompt mode; update yours to include work-mode guardrails.</li>
