## The Big Idea

Uber became the first major company to make the new AI cost problem concrete: 95% of engineers use AI tools monthly, an internal coding agent commits ~1,800 code changes a week — and the COO still cannot connect any of the token spend to better features for customers. Nate's core argument: the token bill is not a verdict on AI. It is information about a kind of work the company started before it built a system to run it. This is a June 2026 problem, not a December 2025 problem.

## His Argument

- **Token burn is information, not waste.** The bill is the first hard evidence that AI has crossed from "a tool you buy" into "labor you have to manage." The company can see usage, commits, and spend — it just lacks a system to translate that labor into legible outcomes. Operating-model failure, not spending failure.
- **The same invoice carries three different threats.** Budget owner: layoff justification. Engineering leader: cap on working experiments. Employee: performance problem. All three readings are wrong because the system — not the spending — is broken.
- **2025 budget controls cannot price agentic work.** Seat licenses and API caps work for discrete queries. They cannot price work that plans, retries, branches, and runs for hours.
- **The answer is in the middle.** Spending freely and capping hard are both easy and both wrong. The durable answer is routing intelligence via [[minimum-effective-intelligence]].

## Receipts

- Uber CTO Praveen Neppalli Naga: company blew through entire 2026 AI budget months early
- 95% of Uber engineers use AI tools monthly (most in agent-style workflows)
- Internal coding agent: ~1,800 code commits per week
- COO Andrew Macdonald: "can see the usage, the commits, and the token spend" but "cannot cleanly connect any of it to better features for customers"
- Named framework: [[minimum-effective-intelligence]] (MEI) — routing rule for frontier vs open vs no-model assignment

## The Contrarian Edge

Consensus: "AI is too expensive, rein in employees." Nate's sharper read: the COO's admission is structural. Uber isn't paying for waste — it's paying for a class of work it hasn't learned to run. The right response is visibility and routing architecture, not a token cap. Companies that respond with spend controls rather than operating-model redesign will repeat this mistake at scale.

## Named Framework

**[[minimum-effective-intelligence]]** (MEI) — A routing principle: assign every AI workload to the minimum intelligence tier (frontier model / open model / no model) that actually meets the task's requirement. Prevents frontier-model spend on commodity tasks and makes token budgets legible.

## What It Means For You

1. Read your token bill as a management signal: decompose into production (outcome-justified), tuition (compounding learning), and waste (broken handoffs). Cap only waste.
2. Apply MEI as a routing rule before any AI investment or tool expansion — tier your stack before the next budget cycle.
3. Redesign work objects (gates, permissions, handoffs) not just access policies — the infrastructure that makes agent spend visible and connectable to outcomes.
4. Audit for the "can't connect to outcomes" gap before the CFO does. Build outcome-tagged workloads and acceptance-rate tracking now.

## Mistake to Avoid

Treating a budget blowup as a cost-discipline failure. Capping token spend without redesigning work-flow leaves you with neither savings nor learning — and suppresses the experiments that were working.
