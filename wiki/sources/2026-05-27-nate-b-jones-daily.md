# I Built a Deck With AI, Then Made a Second AI Attack It.

**Routine:** [[nate-b-jones]] Daily Leader Briefing | **Date:** 2026-05-27
**YouTube:** https://www.youtube.com/watch?v=MFzxIT88zfg
**Substack:** https://natesnewsletter.substack.com/p/ai-office-files-verify-workflow

## Summary

AI can generate a polished PowerPoint or Excel model in minutes — but the output looks finished long before it is trustworthy. The core problem is that models are goal-directed and will guess when sources are missing, producing confident-looking numbers that blend actuals with projections without flagging the distinction.

Nate’s solution is a four-stage workflow: Sources (clean and scoped) → Structure (outline first) → Creation (AI generates in passes) → Verification (hostile reviewer prompt). The hostile reviewer prompt plays the role of a skeptical senior executive and surfaces logical gaps, unsupported claims, and data blending that standard editing misses. The RALPH loop (Codex generates, Opus 4.7 audits) operationalizes this at the model level. The task risk gradient maps where AI is lowest risk (formatting, structure) vs. highest (numerical reasoning, sourcing).

## Key Takeaways

- A prompt asks for output; a workflow defines trust
- Models are goal-directed — they will guess without sources, producing confident-looking errors
- Four-stage workflow: Sources → Structure → Creation → Verification
- The hostile reviewer prompt catches logical gaps and blended data that proofreading misses
- Task risk gradient: formatting (low) → synthesis (medium) → numerical reasoning/sourcing (high)
- The RALPH loop: one model generates, a second audits — weeks per year of productivity upside
