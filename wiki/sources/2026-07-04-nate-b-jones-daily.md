## The Big Idea

Most people will spend this holiday weekend racing to exhaust Fable 5's free tokens — and most will aim them at the wrong work. Fable 5's unique value is as a [[goal-harness]] architect — the layer that defines the objective structure which steers cheaper, specialized models in execution. You don't aim it at the output; you aim it at the goal definition that makes every downstream output stronger.

## His Argument

- **The consensus move is wrong:** "Plan with Fable 5, code with something else" misses the real move. Planning is commodity; designing the *goal harness* — the objective structure that steers the coding model — is the frontier task.
- **The goal harness is the real output:** Not a task list. The set of steering objectives that constrain and guide every downstream model or agent. Fable 5 is exceptionally good at this work.
- **Short prompts beat long ones on hard problems:** Resist over-explaining. Fable 5 performs better with focused, short prompts that force reasoning.
- **Wire it into production tools:** Real leverage comes from integrating into tools the team actually produces output in (e.g., Blender), not advisory chat.
- **Worth paying for — only at the right altitude:** "Fable 5 is worth paying for even after the free tokens are gone, but only if you aim it at problems that genuinely need that horsepower."
- **Broader thesis (Substack):** "We rent the intelligence. We do not own it." Routing skill matters more than model loyalty when any model can be throttled overnight.

## Framework: [[goal-harness]]

The **goal harness** is Nate's term for the objective structure a frontier model like Fable 5 defines — not a task list, but the steering goal-set that constrains and guides whatever model or agent executes the actual work. It separates the goal-definition layer from the execution layer. The harness architect role is what Fable 5 does best; execution goes to cheaper, specialized models.

See also: [[job-first-routing]], [[ai-harness]]

## What It Means For You

1. **Goal-architect this weekend:** Use free Fable 5 tokens to define goal harnesses for your 2–3 most complex ongoing projects — steering objectives that guide AI execution next quarter.
2. **Short-prompt discipline:** Cut prompts to their essential core on hard problems. Added context should improve goal structure, not pad input.
3. **Wire into one production tool:** Move from advisory layer to integrated production-tool layer in one workflow.
4. **Build routing protocol:** Map recurring tasks to daily driver / workhorse / frontier tiers before the next model disruption.

## Mistake to Avoid

Spending frontier tokens on commodity work (summarizing emails, routine copy, standard reports). Nate's cost test: Fable 5 earns its price only on novel domains, unclear problem shapes, judgment-heavy decisions where the goal itself is hard to define. If you can already specify the output, you don't need frontier horsepower.
