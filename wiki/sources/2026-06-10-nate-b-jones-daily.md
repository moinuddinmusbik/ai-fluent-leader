## The Big Idea

Stop asking which is better — Claude Code or Codex. The real question is what each tool is training you to become as an agent operator. Claude trains the *steer* habit: stay close to the work, converse through ambiguity, correct in real time. Codex trains the *dispatch* habit: write a bounded assignment, stand back, demand proof. The skill of 2026 is **[[agent-literacy]]** — not which model wins the benchmark, but which habits you're building.

## His Argument

- **Interfaces train behavior:** Claude and Codex are teaching what an agent is *for*, just as Mac and Windows once taught what a computer was for. The interface you inhabit shapes your mental model, and developers who switch between them report genuine cognitive friction.
- **Claude = Cockpit ([[steer-or-dispatch]] — steer side):** Useful for fuzzy problems where the shape of the question is hard. Tools: CLAUDE.md context files, hooks for automated checks, plan-mode before edits, MCP server connections.
- **Codex = Operations Desk ([[steer-or-dispatch]] — dispatch side):** Parallel threads, visible work queue, inspectable outputs. Key features: a separate 5.5 reviewer model audits execution intent before permitting action; computer use; background automations.
- **Decision rule:** Claude when conversation must precede the assignment; Codex when the task can be written down with sources, files, tools, and checkable proof; both when stakes are high.

## Receipts

- Codex auto-review: a dedicated 5.5 model checks the execution model's intent before permitting action
- Claude 4.8 + workflow mode (new: spins out sub-agents)
- Codex background automations — execute without user present
- Substack paid deep-dive: five agent-run shapes, six pre-launch questions, Run Spec prompt, "is it real?" audit, steer-or-dispatch diagnostic, cross-check prompt

## The Contrarian Edge

- Refuses the "winner" frame: "I don't think the honest answer is Claude wins or Codex wins. They're pulling the future in different directions."
- Nate's own arc: Claude changed his work first, now Codex. Not a dismissal — a phase map for learning.
- Reframes the skill: "Prompting is far too small a word for what we're doing here. We're doing **agent loop management**."
- Real leaderboard: which interface makes permissions obvious, proof hard to forget, repeated work into durable automated skills?

## What It Means For You

1. **Classify the work before picking the tool:** Fuzzy/taste/judgment → Claude. Bounded/assignable/checkable → Codex.
2. **Build proof requirements into every agent run:** Demand receipts (files, logs, diffs, sources) before trusting output.
3. **Use both when stakes are high:** Agent loop management — one plans, one critiques; one implements, one reviews.
4. **Name your failure mode:** Claude = conversation theater (feel close but aren't); Codex = completion theater (run looks done but isn't).

## Mistake to Avoid

Treating tool choice as a one-time benchmark decision. The tool you spend the most time in trains your mental model — and that model outlasts leaderboards. Choose deliberately; learn both ergonomics.
