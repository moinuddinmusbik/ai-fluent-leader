# Shopify CEO Reveals Their Secret AI Developer

**Routine:** [[nate-b-jones]] Daily Leader Briefing | **Date:** 2026-05-26
**YouTube:** https://www.youtube.com/watch?v=NRBQmwlILjk
**Substack:** https://natesnewsletter.substack.com/p/public-ai-work-team-learning

## Summary

Shopify’s River agent is architected to run only in public Slack channels — a deliberate product decision with organizational learning implications. When senior people use AI privately, junior staff see outputs but never see judgment calls, prompt strategies, or the reasoning behind decisions. Over time this widens an apprenticeship gap that traditional mentorship used to close.

Nate invokes Polanyi’s paradox: we know more than we can tell. AI outputs share none of the “how” behind them. The only mechanism for transmitting tacit knowledge at scale is visible process — running real work where others can watch and learn the reasoning, not just the result. Four parts of AI work carry learning value: problem framing, prompt strategy, output evaluation, and the decision to use (or not use) AI output.

## Key Takeaways

- Shopify’s River agent runs in public channels only — visibility is a deliberate design choice
- Private AI usage creates an apprenticeship gap by hiding the reasoning process from junior staff
- Polanyi’s paradox: AI outputs cannot transmit tacit knowledge; only visible process can
- Output-sharing (prompt libraries, polished documents) is not the same as process-sharing
- Regulated teams can use declared spaces with privacy labels to enable visible process safely
