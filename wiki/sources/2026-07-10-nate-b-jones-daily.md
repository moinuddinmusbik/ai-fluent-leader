# Agent-Shaped Work: The One-Minute Agent Test

**Source:** [[nate-b-jones]] | 2026-07-10 | FRAMEWORK | 28:04  
**Video:** [1.6M agents registered for OpenClaw and did NOTHING.](https://www.youtube.com/watch?v=PRqiGS6fnIM)  
**Substack:** [Agent-Shaped Work: When to Use AI Agents (and When Not To)](https://natesnewsletter.substack.com/p/agent-shaped-work) (preview; full post paid)

## The Big Idea

The agents work. What's missing is the dispatch rule. More than 1.6 million AI agents registered on OpenClaw — an agent-only social network — and the overwhelming majority sat idle, never assigned a single task. In China, a paid market emerged to have OpenClaw uninstalled. Nate's argument: this isn't AI failing. It's leaders failing to answer the prior question — *what do I point this at?* That question is a budgeting question, and nobody grew up with instincts for it.

## The Framework: [[one-minute-agent-test]]

Four estimates classify any task in ~60 seconds into one of four verdicts: **Chat · Single Agent · Team of Agents · Don't Bother**.

1. **Size** — Is the task large enough to justify agent setup cost? Small tasks belong in a chat.
2. **Independence** — Can the task run without constant human input? Agents lose advantage if they must pause to ask.
3. **Separation** — Can the work be handed off cleanly? If the judgment for "done right?" is inseparable from the doing, a human owns it. This is the *verification wedge* — one of two hard structural limits.
4. **Checkability** — Can you verify the output mechanically or by sampling? Without it, errors compound silently. The *context ceiling* (the second hard limit) means large agent teams drift without checkpoints.

## How It Works

| Verdict | Condition | Example |
|---------|-----------|---------|
| Chat | Small or judgment-heavy | Back-and-forth brainstorm |
| Single Agent | Large + independent, monolithic output | Research report from bounded corpus |
| Team of Agents | Exceeds context ceiling; parallelizable; verifiable handoffs | $8 website rebuild (~24 agents) |
| Don't Bother | Perfectly agent-shaped but economics fail, or output requires human values | Strategic direction calls |

## Where It Applies (Three Real Tasks)

- **Calendar coordination problem** — fails independence; verdict: Chat or nothing. Coordination is the first work agents absorb — this task is already being automated away.
- **40-tool audit** — scores on all four; verdict: Team. Paid for itself. Retiring failing tools covered the agent run cost.
- **Judgment call** — separation fails; the judgment *is* the task. Nate kept it. Frontier models still lose to domain experts on value-laden questions.

## Receipts

- **1.6M idle agents:** 1.6M+ registered on OpenClaw; most never dispatched once.
- **China uninstall market:** Paid market emerged to remove a free tool — people couldn't figure out what to point it at.
- **Stanford (15.9% → 56%):** Cheap model brute-forced to 3.5× better accuracy by spending more tokens, not by switching models.
- **Anthropic (80% variance):** Token spend explained 80% of performance gap between strong and weak agent runs.
- **Two structural limits:** The *verification wedge* and the *context ceiling* — every multi-agent pitch collapses to one of these.
- **250 attempts:** Nate validated the four-category framework across 250 test runs before publishing.
- **40-tool audit:** Audit paid for itself in tool retirements.

## The Contrarian Edge

The market sells more agents. Nate sells the "Don't Bother" verdict — the one nobody else gives you. A task can be perfectly agent-shaped and still fail the economics test. Two dials decide: build cost + run cost. Ignoring them is how you join the 1.6M idle installs.

## What It Means For You

1. **Run four estimates before every AI assignment.** Size → Independence → Separation → Checkability. Fail two? Default to chat.
2. **Spend on token budget, not on model upgrades.** Stanford's 3.5× jump and Anthropic's 80% finding both point at the same lever.
3. **Commission a tool audit this quarter.** Apply the four estimates to every AI tool installed. Retire what fails. It's agent-shaped work — it should pay for itself.
4. **Name your judgment calls explicitly.** Write the list before a vendor convinces you to automate them.

## Mistake to Avoid

Installing an agent and never dispatching it. The failure mode isn't the AI misbehaving — it's the AI sitting idle because nobody answered the prior question: *what do I point it at?* 1.6 million people made this mistake simultaneously.
