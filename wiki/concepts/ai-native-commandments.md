# AI-Native Commandments

Nate B. Jones's 15-commandment operating system for moving a company from tool adoption to AI-native speed. Introduced in [[2026-07-12-nate-b-jones-daily]].

## Definition

A set of 15 deliberately severe rules written in "thou shalt not" language that together form one operating system — not a menu — for organizational transformation in the AI era. Each commandment exists because AI has changed the work underneath it. They cannot be extracted individually; the system only works when all commandments move together.

## The Four Clusters

**Speed (C1, C3–6):** Thou shalt not slow down engineering; product does not make roadmaps; product does not control engineering time; product in code daily; product sits with engineering daily.

**Time/Meetings (C2, C7):** No meeting longer than one hour; kill monthly meetings.

**Systems (C11–12):** Documentation as code (agent-readable); design into code, SDK, and UI.

**Human (C13–15):** Stay flexible; assume best intent; build with teams; the teaching commandment.

## The Replacement Rule

The most important operational insight: killing a commandment's target (e.g., roadmaps) only works if other commandments absorb the coordination job the target was doing. C3 (no roadmaps) requires C4 (product in code daily) + C6 (product sits with engineering daily) to actually function. Ban without replacement = deleted coordination.

## The Enforcement Mechanism — [[enforcement-ladder]]

Commandments run as software: value → instruction → reminder → hard block → human-owned decision. Machine asks; person keeps judgment.
