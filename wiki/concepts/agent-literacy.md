# Agent Literacy

A [[nate-b-jones]] framework for the core competency of 2026: the ability to effectively manage AI agents — knowing when to steer, when to dispatch, and when to verify.

## Definition

Agent literacy is not prompting (Nate: "prompting is far too small a word for what we're doing here"). It is **agent loop management**: writing assignments that come back as inspected work, demanding proof before trusting output, and building the judgment to know when work is good enough to leave the machine.

## Core Components

1. **Steer or dispatch** — knowing which mode each task demands (see [[steer-or-dispatch]])
2. **Assignment design** — turning fuzzy goals into bounded tasks with clear proof criteria
3. **Proof requirements** — always demanding receipts (files, logs, diffs, sources) before trusting what an agent returns
4. **Failure mode awareness** — knowing the failure mode of your tool: conversation theater (Claude) or completion theater (Codex)
5. **Loop management** — using multiple agents: one plans, one critiques; one implements, one reviews

## Why "2026 skill"

Coding agents arrived in software first because code has built-in proof ("does the code run or not?"). The same habits are now spreading to all knowledge work — research, legal, finance, operations. "A chatbot answers and an agent takes a job." Leaders who build agent literacy now will have a durable advantage as this spreads.
