# Imagination Gap

## Definition

The **imagination gap** is Nate B. Jones's term for the primary competitive divide in AI-enabled organizations: not which models teams use or how cheaply they run them, but whether leaders are asking for *new* work that cheap execution now makes possible, or simply doing the *old* work faster and cheaper.

## Key Aspects

- **Not a tools gap.** The gap is not in model access or even routing discipline — those are increasingly universal. It is in the organizational capacity to identify work worth doing that previously couldn't be attempted.
- **Triggered by execution becoming cheap.** When AI execution costs collapse (e.g., Fable 5 at $10/M input), the constraint shifts from "can we afford to run this?" to "do we have the imagination to ask for it?"
- **Evidenced by the $40 experiment.** Mitchell Hashimoto's $40 job — a task no routing table would assign — produced differentiated results precisely because it was outside the optimization envelope. Advantage lives there.
- **Organizational, not individual.** The diagnostic question is not "does someone in this org have imagination?" but "does the organizational agenda include work that wasn't possible 18 months ago?" If the AI task list looks identical to the pre-AI task list (just faster), the imagination gap is present.

## The $40 Diagnostic

Nate's one-question test: *"What would you attempt if AI execution cost $40?"* If nobody in the org has an answer, the bottleneck is imagination, not model price.
