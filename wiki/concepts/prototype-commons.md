# Prototype Commons

The **abundance of AI-built software assets** that accumulates inside an organization when anyone can build a working prototype in an afternoon — creating hidden demand and hidden risk simultaneously.

## The New Reality

Microsoft has over 1 million power platform assets built internally (2026). The artifact arrives before the request. Traditional product management — which gatekept what got built — has been bypassed. The question is no longer “Should we build this?” but “Somebody already built something — should the company rely on it?”

## The Production Class Ladder

The organizational response to the prototype commons is a production class ladder with explicit criteria and demotion authority:

| Class | Definition | Who relies on it |
|-------|-----------|------------------|
| **Prototype** | Built, not validated, not relied on | Creator only |
| **Tested** | Validated for limited use cases | Defined team, known scope |
| **Production** | Full reliance, maintained, monitored | Organization-wide |

**Demotion** (moving an asset down the ladder) matters as much as promotion. Unqualified reliance on Prototype-class assets is the hidden risk.

## The New PM Decision Rule

> “Somebody already built something — should the company actually rely on it?”

This replaces the traditional PM question (“Should we build this?”) as the primary product judgment call. Market judgment over prototyping speed is the new scarce skill.
