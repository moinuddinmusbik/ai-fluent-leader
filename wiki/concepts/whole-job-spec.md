# Whole-Job Spec

A nine-field framework introduced by [[nate-b-jones]] for converting an AI prompt into a whole-job delegation to a frontier model. Introduced in the Substack post paired with the 2026-06-30 Short; first hinted at in the 2026-06-23 long-form episode on Fable 5.

## Core Idea

The Whole-Job Spec forces you to think in delegations, not instructions. Before handing any significant job to a frontier model, Nate fills out a structured spec with nine fields that define:

- **Job definition** — what the job actually is, in one sentence
- **Start state** — what exists before the model begins
- **Success criteria** — what done looks like, concretely
- **Failure modes** — what the model should NOT do or decide
- **Review trigger** — where the model should stop and hand back to you
- **Output format** — how results should be structured
- (Plus three additional fields in the paid guide)

## Why It Matters

Without a spec, users default to prompt mode: one-step instructions that produce one-step outputs. The Whole-Job Spec shifts the unit of work from a prompt to a run — the same shift Nate describes as moving from "prompt mode" to "work mode" (see also [[open-engine]] from 2026-06-26).
